# 실행 계획 (Exec Plans)

복잡한 작업의 계획, 진행 상황, 의사결정 기록을 버전 관리한다.

## 구조

```
exec-plans/
├── active/      # 진행 중인 계획
├── completed/   # 완료된 계획
└── README.md
```

## 파일 네이밍

```
YYYY-MM-DD-<kebab-case-title>.md
예: 2026-04-03-auth-refactor.md
```

## 계획 파일 템플릿

```markdown
# [제목]

**날짜**: YYYY-MM-DD
**상태**: active | completed | cancelled
**담당**: (선택)

## 목표

무엇을 달성하려는지 1-3문장.

## 배경 / 동기

왜 이 작업이 필요한지.

## 접근 방법

단계별 실행 계획.

- [ ] 단계 1
- [ ] 단계 2
- [ ] 단계 3

## 의사결정 로그

작업 중 내린 중요한 결정과 이유.

## 결과 (완료 시)

무엇이 달성되었는지.
```

## 원칙

- 계획은 일급 아티팩트다. 코드 변경과 함께 커밋한다.
- 완료된 계획은 `active/`에서 `completed/`로 이동한다.
- 취소된 계획도 보존하고 이유를 기록한다.
- 에이전트는 작업 시작 전 `active/`를 확인하여 진행 중인 계획을 파악한다.
